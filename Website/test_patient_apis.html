1. Check Active Patient

Returns the row ID of active patient (done=0) for an ambulance

2. Update Patient Vitals

Updates sensor data for an existing patient record

3. Database Quick Actions

SQL queries for testing (copy to phpMyAdmin)

Create Active Patient:

INSERT INTO patients 
(ambulance_id, patient_id, patient_name, hospital, done)
VALUES 
('AMB-001', 'P123', 'Test Patient', 'Hospital A', 0);

Mark Patient as Done:

UPDATE patients 
SET done = 1 
WHERE id = 1;

View Active Patients:

SELECT id, ambulance_id, patient_id, 
       patient_name, hospital, done 
FROM patients 
WHERE done = 0;

4. ESP32 Workflow Test

Simulate ESP32 behavior

Scenario 1: No Active Patient

  1. Ensure no patients with done=0 for AMB-001
  2. Click "Check Active Patient"
  3. Should return patient_row_id = 0
  4. ESP32 shows "No Patient" on OLED

Scenario 2: Active Patient Found

  1. Create patient using SQL above
  2. Click "Check Active Patient"
  3. Should return patient_row_id > 0
  4. Use that ID to "Update Vitals"
  5. ESP32 reads sensors and uploads

Scenario 3: Patient Marked Done

  1. Mark patient as done using SQL
  2. Try to "Update Vitals"
  3. Should fail with "marked as done"
  4. ESP32 stops updating, waits for new patient
